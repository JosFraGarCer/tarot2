# ❌ Decisiones Descartadas

> **Propósito:** Registro de ideas que fueron rechazadas y por qué
> **Valor:** Evitar repetir errores, documentar razonamiento

---

## Por qué Documentar Rechazos

1. **Memoria institucional**: Saber qué ya se consideró
2. **Evitar ciclos**: No rediscutir lo mismo
3. **Aprendizaje**: Entender qué no funciona
4. **Transparencia**: Mostrar el proceso de diseño

## Descartadas

### DES-001: Sistema de Dados d20

| Campo | Valor |
|-------|-------|
| **Propuesta original** | Usar 1d20 como sistema base |
| **Fecha consideración** | Octubre 2024 |
| **Razón de rechazo** | Demasiado asociado a D&D, menos narrativo |
| **Alternativa elegida** | 2d12 (Habilidad + Destino) |
| **Estado** | ❌ DESCARTADA |

**Análisis:**
- El d20 tiene distribución plana (5% cada número)
- No permite la mecánica del Giro del Destino
- Menos distintivo para la identidad del juego

## Proceso de Descarte

### Criterios para Descartar

1. **Pruebas fallidas**: 2+ tests con resultados negativos
2. **Contradicción**: Va contra filosofía del juego
3. **Complejidad**: Añade más de lo que aporta
4. **Alternativa mejor**: Existe solución superior

### Al Descartar

1. Documentar en este archivo
2. Incluir razón clara
3. Mencionar alternativa elegida
4. Actualizar EN-PRUEBAS.md si estaba ahí

## Lecciones Aprendidas

| Área | Lección |
|------|---------|
| Dados | Menos dados = más rápido |
| Combate | PA bajos = combates cortos |
| Magia | Necesita coste para balance |
